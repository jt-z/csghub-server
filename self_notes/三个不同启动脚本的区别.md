## 三个训练脚本的区别与主要逻辑

### 核心共同点

三个脚本**最终都调用相同的训练命令**：
```bash
CUDA_VISIBLE_DEVICES=0 llamafactory-cli train ...
```
目标都是执行 **SFT（监督微调）训练**，使用 LoRA 方法。

---

### 主要区别对比

| 特性 | `train_sft.sh` | `train_sft_full.sh` | `train_with_yaml.sh` |
|------|----------------|---------------------|----------------------|
| **参数传递方式** | 命令行参数 | 命令行参数 | YAML配置文件 |
| **参数完整性** | 完整参数 | 完整参数 | 完整参数 |
| **可定制性** | 通过环境变量 | 通过环境变量 | 修改YAML文件 |
| **灵活性** | 中等 | 中等 | 高 |
| **可读性** | 一般 | 一般 | 高 |
| **版本控制** | 差 | 差 | 好 |
| **适用场景** | 快速测试 | 快速测试 | 生产环境/复杂配置 |

---

### 1. train_sft.sh（基础训练）

**主要逻辑**：
- 通过**环境变量**接收定制参数（MODEL_NAME, DATASET, TRAIN_EPOCHS, LEARNING_RATE, OUTPUT_DIR）
- 所有训练参数通过**命令行**直接传递给 `llamafactory-cli train`
- 最后一行有 `--use_unsloth false` 参数

**使用方式**：
```bash
# 默认参数
./train_sft.sh

# 自定义参数
MODEL_NAME=Qwen/Qwen2-0.5B DATASET=phone_en TRAIN_EPOCHS=5 ./train_sft.sh
```

---

### 2. train_sft_full.sh（完整参数）

**主要逻辑**：
- 与 `train_sft.sh` **几乎完全相同**
- 通过**环境变量**接收定制参数
- 所有训练参数通过**命令行**传递
- **唯一区别**：没有 `--use_unsloth false` 参数

**使用方式**：
```bash
MODEL_NAME=Qwen/Qwen2-0.5B DATASET=phone_en ./train_sft_full.sh
```

---

### 3. train_with_yaml.sh（YAML配置）

**主要逻辑**：
- 通过**YAML配置文件**传递所有训练参数
- 默认使用 `/etc/csghub/example_sft.yaml`
- 可通过 `CONFIG_FILE` 环境变量指定其他配置文件
- **核心命令非常简洁**：
  ```bash
  llamafactory-cli train ${CONFIG_FILE}
  ```

**使用方式**：
```bash
# 使用默认配置
./train_with_yaml.sh

# 使用自定义配置
CONFIG_FILE=/path/to/your/config.yaml ./train_with_yaml.sh
```

**YAML配置文件结构**（`example_sft.yaml`）：
```yaml
# 模型配置
model_name_or_path: Qwen/Qwen2-0.5B
template: qwen2
finetuning_type: lora

# 训练配置
stage: sft
do_train: true
num_train_epochs: 3
learning_rate: 1.0e-5

# LoRA配置
lora_rank: 8
lora_alpha: 16
lora_dropout: 0.05

# 输出配置
output_dir: /workspace/output
```

---

### 参数对比表

三个脚本使用的核心参数完全一致：

| 参数类别 | 具体参数 |
|----------|----------|
| **模型配置** | `model_name_or_path`, `template`, `finetuning_type` |
| **数据集** | `dataset`, `dataset_dir`, `cutoff_len` |
| **LoRA参数** | `lora_rank`, `lora_alpha`, `lora_dropout`, `lora_target` |
| **训练超参数** | `num_train_epochs`, `per_device_train_batch_size`, `gradient_accumulation_steps`, `learning_rate`, `weight_decay`, `lr_scheduler_type`, `warmup_ratio` |
| **输出配置** | `output_dir`, `logging_steps`, `save_steps`, `save_total_limit` |
| **优化配置** | `bf16`, `gradient_checkpointing`, `max_grad_norm` |

---

### 选择建议

| 场景 | 推荐脚本 |
|------|----------|
| **快速测试/验证** | `train_sft.sh` 或 `train_sft_full.sh` |
| **简单定制** | `train_sft.sh`（通过环境变量） |
| **复杂配置/生产环境** | `train_with_yaml.sh`（YAML文件更易管理） |
| **版本控制** | `train_with_yaml.sh`（配置文件易于Git管理） |
| **多实验对比** | `train_with_yaml.sh`（多个YAML配置文件） |
